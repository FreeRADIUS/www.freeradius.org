EAP Methods

FreeRADIUS was the first Open Source RADIUS server to support EAP. It has defined the standard for how RADIUS servers should manage EAP sessions. As of Version 2.0, it supports more EAP methods than any other RADIUS server, commercial or Open Source.

The following EAP methods are supported by FreeRADIUS 2.0 and later for wired, or for WiFi authentication.

  • EAP-AKA1
  • EAP-FAST1
  • EAP-GPSK1
  • EAP-IKEv2 3
  • Cisco LEAP2
  • EAP-PAX1
  • EAP-PEAPv0
    • EAP-MSCHAPv23
    • EAP-GPSK1
    • EAP-GTC3
    • EAP-MD5-Challenge3
    • EAP-PAX1
    • EAP-PSK1
    • EAP-SAKE1
    • EAP-TLS2
  • EAP-PEAPv11
    • EAP-MSCHAPv2
    • EAP-GPSK
    • EAP-GTC
    • EAP-MD5-Challenge
    • EAP-PAX
    • EAP-PSK
    • EAP-SAKE
  • EAP-PSK1
  • EAP-SAKE1
  • EAP-SIM3
  • EAP-TLS3
  • EAP-TTLS
    • PAP3
    • CHAP3
    • MS-CHAP3
    • MS-CHAPv23
    • EAP-MSCHAPv23
    • EAP-MD53
    • EAP-GPSK1
    • EAP-GTC3
    • EAP-PAX1
    • EAP-PSK1
    • EAP-SAKE1
    • EAP-TLS2
  • EAP-TNC2

The following methods are also supported, but cannot be used with WPA or IEEE 802.1X WEP keying.

  • EAP-MD5-Challenge3
  • EAP-MSCHAPv23
  • EAP-GTC3

Notes

1   Considered stable in FreeRADIUS 3.0 and later. In FreeRADIUS 2.x only via the "eap2" module, by using the library libeap.so from hostapd. This module was available in FreeRADIUS versions 2.x and and removed with the release of version 3.0.0. "eap2" was considered experimental and not production ready.

2   Via the "eap" module, using the native FreeRADIUS EAP implementation.

3   Supported via the "eap" module with FreeRADIUS 3.0 and later. In FreeRADIUS >= 2.0 < 3.0 both the "eap" and "eap2" could beused but some of their functionality may have been different between the two implementations. See their configuration files for more details.
